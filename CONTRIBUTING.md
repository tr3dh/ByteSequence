# Contributing

## How to contribute

If you want to develop a feature, fix something, or take the project in a new direction — fork it. You're free to build on this however you like within the bounds of the license.

If your changes are small, focused, and clearly useful, feel free to open a pull request. There's no guarantee of a merge or a timeline, but reasonable contributions will be read and considered.

## Before opening a PR

- Open PRs only for small seperate Changes
- For large features or fixes discuss it in an issue first
- Make sure the code works

## In short

Feel free to fork the project, open dissusions or pull requests for concrete improvements, but dont expect an immediate review or a guaranted merge.
